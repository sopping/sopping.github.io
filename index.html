什么是函数式编程

函数式编程（英语：functional programming）或称函数程序设计、泛函编程，是一种编程范式，它将计算机运算视为函数运算，并且避免使用程序状态以及易变对象。

  • Haskell
  • Scala：基于jvm

指令式编程（英语：Imperative programming）；是一种描述计算机所需作出的行为的编程典范。几乎所有计算机的硬件工作都是指令式的；几乎所有计算机的硬件都是设计来运行机器代码，使用指令式的风格来写的…每步都是指令。

  • Java

描述数学表达式 x = (1 + 2) * 3 - 4

把运算过程尽量写成一系列嵌套的函数调用

没有副作用

不修改状态

纯粹的函数式编程语言通常不包含while或for，使用递归代替迭代，“状态”保存在变量中。

引用透明-纯函数

如果程序中任意两处具有相同输入值的函数调用能够互相置换，而不影响程序的动作，那么该程序就具有引用透明性

任何时候只要参数相同，引用函数所得到的返回值总是相同的。

  • 缓存计算结果（Memorization）

  • 延迟求值（Lazy Evaluation）

RxJava的线程切换

发表于 2018-07-02 更新于 2019-09-01 分类于 RxJava

在使用RxJava时，我们经常会用到线程切换方法observeOn和subscribeOn。当存在多个subscribeOn和observeOn的时候，线程怎么切换的？如何确定生产者、各个操作符、消费者分别是在哪个线程执行呢？

  1. 生产者在哪个线程产生数据
  2. 4个map变换分别在哪个线程执行
  3. doOnSubscribe在哪个线程执行
  4. 消费者在哪个线程执行消费数据

本文使用的RxJava版本为2.1.13

RxJava调用控制过程

RxJava的调用链分为两个阶段，这里借助网上资料的流程说明下

  1. 驱动阶段。整个异步事件流的触发由subscribe开始。它发起了一个反向驱动过程（从下游到上游），跨过每一个中间的Observable和OnSubscribe，到达第一个Observable（产生事件的源头）。对应图中的(1)和(2)。这个阶段一般就是从下游到上游调用一次就结束了。
  2. 事件发射阶段。第一个Observable开始产生事件，然后事件流就开始正向传递，经过每一个中间的Observable，最终到达Subscriber（事件的消费者）。对应图中的(3)。与前一阶段不同，事件从上游往下游传递，不是一次就完了，而是多个事件组成的事件流。

总结下：
每个subscribeOn都会生成一个ObservableSubscribeOn对象，驱动阶段中，这些对象的subscribe都在当前指定的scheduler调度器上执行，直到追溯到上游尽头的生产者。所以，生产者的subscribe方法（即生成消息方法），是在距离生产者最近的第一个scheduler调度器上执行的。

指定观察者将在哪个调度器上观察

source.subscribe在当前线程执行，故不影响驱动阶段。看下ObserveOnObserver，这里关注事件发射的onNext方法

总结下：
observeOn缓存上游消息，然后在指定线程上调度执行下游的onNext方法。故它影响的是下游消息转换和消费执行线程

doOnSubscribe

其实是一个生命周期时间，类似于Spring的AOP，返回的是一个ObservableDoOnLifecycle对象

实现很简单，在当前线程调用source.subscribe

总结：doOnSubscribe不会切换线程，subscribe触发的驱动阶段到达时，由当前source所处的调度器决定所在线程

组合

官方文档给出了observeOn/subscribeOn同时使用时的线程转换示意图

  • subscribeOn指定了生产者产生数据的线程
  • observeOn改变了下游操作符(map)的执行线程，直到下一个observeOn
